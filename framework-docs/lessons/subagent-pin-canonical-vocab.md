## 실패: 서브에이전트에 정본 어휘·열거값을 안 박고 형식만 전달
- 시도한 것: `docs-lint` 스킬을 서브에이전트로 작성. "2축 형식 + lint op"라는 요구만 주고 정본 action 어휘를 프롬프트에 명시하지 않았다.
- 무엇이 망했나: 서브에이전트가 action 어휘를 `fix/archive/merge/split/orphan`으로 **임의 생성**했다 — 설계의 정본(`create/update/delete`)과 불일치. 컨트롤러 리뷰에서 별도 커밋으로 정정해야 했다.
- 제약(왜): 서브에이전트는 우리 대화 맥락이 없고, 빈칸을 *그럴듯하지만 벗어난* 값으로 채운다. 형식 *틀*만 주면 세부 어휘는 발명된다.
- 회피·우회: 정본 어휘·열거값(허용 action, op 집합, 파일명 규칙 등)을 프롬프트에 **명시 열거**한다. 또는 컨트롤러가 청크 리뷰에서 정본과 대조해 정정한다([[team-design-review]]).
