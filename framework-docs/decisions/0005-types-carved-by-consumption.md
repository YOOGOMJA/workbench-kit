## 결정: docs 페이지 유형의 카빙 기준
- 상태: 채택
- 선택: 페이지 유형은 *쌓는 구조*가 아니라 **소비(시작 task의 질문)**가 카빙한다. 결정·실패를 1급 폴더로(소비 진입점 1순위), `모델` 엔티티는 제거.
- 맥락: 시작하는 task가 가장 묻고 싶은 건 "이미 정했나?/망했나?". 기법·도구·개념은 이미 skills·AGENTS·용어집이 소비를 처리한다.
- 이유: 쌓는 구조 ≠ 소비 구조이고 후자가 이긴다. 결정·실패는 소비 표면(앵커 index)이 따로 없던 유일한 1급 질문이었다. `모델`은 워크벤치 도메인에 대상이 없는 Karpathy LLM-위키의 잔재.
- 기각한 대안: ① 엔티티 5종(기법/도구/모델/결정/실패)을 쌓기 기준으로 평면 나열 → 축이 섞이고 모델 서랍이 빈다. ② 결정·실패를 provenance로 강등 → 주제를 모르는 시작 task가 소비 못 함.
